# AV-Forge 2.0 — Data Curation Strategy

The Seedance 2.0 report does not disclose data. This document specifies a curation strategy **targeted at the fine-grained categories** in the report's Tables 3–8, 11–23, 26–28, so that AV-Forge 2.0 reaches the targets in `01_DESIGN.md` §1.4.

The principle: **every fine-grained evaluation category must have a dedicated, balanced data slice with explicit coverage targets**, and the weakest competitor categories (where Seedance 2.0 already wins) get **over-sampled hard negatives** so AV-Forge surpasses them.

---

## 1. Data sources (tiers)

| Tier | Source | Use |
|---|---|---|
| A — Licensed professional | cinematic/TV VFX libraries, ad archives, game cinematics, opera/dialect recordings, multilingual singing | high-quality targets, aesthetics, cinematography, audio expressiveness |
| B — Public large-scale | open video corpora (HD video, multilingual speech, music, ambient/SFX libraries) | backbone pretraining, breadth |
| C — Synthetic | generated by AV-Forge 1.x + competitors (with licenses) + procedural VFX/sim | hard negatives, physics edge cases, counter-reality, surreal |
| D — Expert-curated | shot-detected + camera-grammar-tagged professional footage | shot-planner supervision, cinematography reward |
| E — Reference pairs | (subject image, video of subject), (style image, video in style), (motion video, target), (audio clip, video with that audio) | reference-task training |

All data passes the safety pipeline in §6 (CSAM/identity/unwilling-likeness removal at ingest; **no content-type filtering** — adult/edgy/spicy content is retained for creative use, only non-consensual/illegal material is removed).

### 2.5 Audio-Visual Sync (Table 7) — 17 categories
Same slices as 2.4, but **paired with aligned video** and supervised with forced-aligner + SyncNet labels. Hard negatives: temporally shifted audio (±2–5 frames) so the model learns sub-frame alignment. Multi-speaker scenes get per-speaker phoneme alignment (fixes the report's lip-sync weakness).

### 2.6 Audio Prompt Following (Table 8) — 17 categories
Same slices; prompts specify timbre, language, dialect, instrument, spatialization. Hard: prompts that specify *combinations* (e.g., "Sichuan-dialect narration over binaural rain with a guzheng motif").

## 3. Curriculum & mixing

- **Stage 1 (pretrain):** 70% B, 20% A, 10% C. Duration 4s, 480p.
- **Stage 2 (multimodal):** 50% A, 25% E, 15% C, 10% B. Duration up to 15s, 720p.
- **Stage 3 (continuation/extension):** 60% continuation/extension slices, 40% general.
- **Stage 4 (alignment):** reward-weighted sampling; hard-negative up-weighted 3×.

Per-batch **category balancing**: a sampler ensures every fine-grained category appears in proportion to its target volume, with a minimum floor so rare categories (opera, minority languages, ASMR) are never starved.

---

## 4. Annotation pipeline

- **Shot detection + camera grammar** (Tier D): automatic shot boundary + camera-move classification + 180°-rule checker → shot-plan supervision.
- **Forced alignment** (audio): multilingual ASR + forced aligner → per-phoneme timestamps → lip-sync + AV-sync labels.
- **Speaker/entity IDs**: face/voice clustering → stable slot IDs per clip.
- **OCR** on text-bearing frames → text-region masks + transcripts → text-rendering reward.
- **Physics tags**: contact/support/momentum labels via a lightweight physics estimator (and sim data has ground-truth).
- **Palette tokens**: color-histogram per clip → palette embedding → continuation color-consistency reward.
- **Aesthetics scores**: human-preference model trained on pairwise labels from expert evaluators (ad/game/VFX pros, per the report's SeedVideoBench 2.0 protocol).
- **Safety tags**: CSAM/identity/unwilling-likeness classifiers at ingest (output-boundary only; no content-type filtering).

---

## 5. Synthetic hard-negative generation

To surpass Seedance 2.0 on its *own* weak spots, generate hard negatives targeting the report's acknowledged weaknesses:
- **Deformation clusters**: procedurally perturb limb/face meshes → negative physics reward.
- **Multi-speaker lip-sync**: multi-talker scenes with overlapping speech → per-speaker alignment reward.
- **Text restoration in edits**: edit prompts that change text in-frame → OCR reward on masked region.
- **Extension seam**: source→extension pairs with deliberate color/identity drift → palette + count rewards.
- **HF noise**: add structured HF noise → spectral loss.

---

## 6. Safety & ethics at the data layer

- **Ingest filtering:** CSAM, non-consensual intimate imagery, unwilling real-person likeness → removed. This is the *only* content filter.
- **No content-type filtering:** adult/edgy/spicy/violent/surreal creative content is **retained** and used for training, per the spec's "no over-refusal" requirement.
- **Provenance:** all clips carry licensing + provenance metadata; generated outputs carry a watermark.
- **Bias auditing:** per-category demographic balance reporting (for fairness in subject reference, not for refusal).
